HLW: Code

This site is built with Hakyll, which uses pandoc to convert markdown documents to html. One nice thing about pandoc is the ability to make your own filters, i.e. Pandoc → Pandoc transformations that can pass through other interesting Haskell-definable structures. In other words, we can convert the strings in the document into structured data! So I wrote a little filter that takes blocks labelled lemmon and creates proof structure out of them, i.e. a list of proof lines, where each proof line is a triple consisting of reference numbers, formula, and justification.

The input is expected to look like a manually typed proof in Lemmon-Suppes format:

```lemmon
1    (1) ¬P                       A
2    (2) Q                        A
1,2  (3) ¬P∧Q                     1,2 ∧I
4    (4) (¬P∧(Q∨R))→¬(S∧(¬T∨U))   A
```

The parser expects whitespaces between the three columns, and it tries to guess that the second column is by looking for the line number in parentheses. If the formula after the line number is well-formed, it converts it into type PropFormula. (Yes, I still need to extend parsing to quantifier logic.)

There are all sorts of interesting things we could do now. For example, we could ask what the main connectives are of the formulas on the lines. Or we could ask if a lemmon block represents a valid proof.
